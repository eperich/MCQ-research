<zyHTML id="8cee9911-9ab9-dd72-7ee0-a5eb70f1bbcb" subsection="true" caption="Comments">
<h3>Comments</h3>
</zyHTML>


<zyHTML id="1601aed8-e727-4b83-8e34-9adaee3ee16d">
<p><zyDefn>A _comment_  is text a programmer adds to code, to be read by humans to better understand the code but ignored by the compiler.</zyDefn> Two common kinds of comments exist: </p>

<ul>
<li><zyDefn>A _single-line comment_ starts with // and includes all the following text on that line.</zyDefn> Single-line comments commonly appear after a statement on the same line. </li>
<li><zyDefn>A _multi-line comment_ starts with /* and ends with */, where all text between /* and */ is part of the comment.</zyDefn> <zyDefn> A multi-line comment is also known as a _block comment_.</zyDefn></li>
</ul>
</zyHTML>




<zyVersion version="java">
<zyFigure caption="Comments example." id="a3c38dad-91c6-4c4a-6a7a-8239e5e0db79">
<zyCode language="java" numbering="true">
import java.util.Scanner;

/*
   This program calculates the amount of pasta to cook, given the
   number of people eating.

   Author: Andrea Giada
   Date:   May 30, 2017
*/

public class PastaCalculator {
   public static void main (String [] args) {
      Scanner scnr = new Scanner(System.in);
      int numPeople;         // Number of people that will be eating
      int totalOuncesPasta;  // Total ounces of pasta to serve numPeople

      // Get number of people
      System.out.println("Enter number of people: ");
      numPeople = scnr.nextInt();

      // Calculate and print total ounces of pasta
      totalOuncesPasta = numPeople * 3;  // Typical ounces per person
      System.out.println("Cook " + totalOuncesPasta + " ounces of pasta.");
   }
}
</zyCode>
</zyFigure>
</zyVersion>

<zyQ>

<zyCode>/* Get user input */</zyCode></zyQtext>
<zyQexpl>A multi-line comment is allowed on a single line, but good practice is to use // for single-line comments, reserving /* */ for multi-line comments. </zyQexpl>
</zyQ>

<zyQ>
<zyQtext correct="Valid" truelabel="Valid" falselabel="Invalid">
<zyCode>/* Determine width and height,
   calculate volume, 
   and return volume squared.
*/</zyCode></zyQtext>
<zyQexpl>A typical multi-line comment.</zyQexpl>
</zyQ>

<zyQ>
<zyQtext correct="Valid" truelabel="Valid" falselabel="Invalid">
<zyCode>// Print "Hello" to the screen // </zyCode></zyQtext>
<zyQexpl>The second // is ignored just like other text after the first //. </zyQexpl>
</zyQ>

<zyQ>
<zyQtext correct="Invalid" truelabel="Valid" falselabel="Invalid">
<zyCode>// Print "Hello"
   Then print "Goodbye"
   And finally return.
//</zyCode></zyQtext>
<zyQexpl>The first line's // only applies to the remainder of the line. The second and third lines would yield compiler errors. </zyQexpl>
</zyQ>

<zyQ>
<zyQtext correct="Valid" truelabel="Valid" falselabel="Invalid">
<zyCode>/*
 * Author: Michelangelo
 * Date: 2014
 * Address: 111 Main St, Pacific Ocean
 */</zyCode></zyQtext>
<zyQexpl>Programmers commonly use single asterisks to make clear which lines are contained in the comment. Those single asterisks have no special meaning; they are just text within the /* ... */. </zyQexpl>
</zyQ>

<zyQ>
<zyQtext correct="Valid" truelabel="Valid" falselabel="Invalid">
<zyCode>// numKids = 2;  // Typical number </zyCode></zyQtext>
<zyQexpl>The entire line after the first // is ignored. The numKids = 2 statement won't execute, but the code is valid. (Perhaps the programmer intentionally commented out the code for the moment). </zyQexpl>
</zyQ>

<zyQ>
<zyQtext correct="Valid" truelabel="Valid" falselabel="Invalid">
<zyCode>/* 
   numKids = 2;  // Typical number
   numCars = 5; 
*/ </zyCode></zyQtext>
<zyQexpl>The programmer has likely temporarily "commented out" those two statements. The // within the /* .. */ will be ignored,  like the other text. </zyQexpl>
</zyQ>

<zyQ>
<zyQtext correct="Invalid" truelabel="Valid" falselabel="Invalid">
<zyCode>/* 
   numKids = 2;  /* Typical number */
   numCars = 5;
*/</zyCode> </zyQtext>
<zyQexpl>The first /* starts a multi-line comment. That comment ENDS at the */ on the SECOND line. The compiler will compile numCars = 5, then generate an error when reaching a */ that isn't ending a comment. Programmers use // when possible in part so that /* ... */ can be used to "comment out" code.   </zyQexpl>
</zyQ>

<zyHTML id="7514cabe-1a4f-c236-ccf7-8b3ea6c2d2ab" subsection="true" caption="Whitespace">
<h3>Whitespace</h3>
</zyHTML>


<zyHTML id="63044e73-ae9b-41e0-8729-cda05db677ca">
<p><zyDefn> _Whitespace_ refers to blank spaces (space and tab characters) between items within a statement and blank lines between statements (called newlines).</zyDefn> A compiler ignores most whitespace.  </p> 

<p><zyGoodPractice id="de73e843-dbd0-ece5-36c2-cf59d5a4312a">Good practice is to deliberately and consistently use whitespace to make a program more readable.</zyGoodPractice> Programmers usually follow conventions defined by their company, team, instructor, etc., such as:</p>

<ul>
<li>Use blank lines to separate conceptually distinct statements. </li>
<li>Indent lines the same amount.</li>
<li>Align items to reduce visual clutter. </li>
<li>Use a single space before and after any operators like =, +, *, or / to make statements more readable. </li>
</ul>
</zyHTML>



<zyVersion version="java">
<zyFigure caption="Good use of whitespace." id="348cd661-75bf-d59b-23ab-c5488e51b56f">
<zyCode language="java" numbering="true">
import java.util.Scanner;

public class WhitespaceEx {
   public static void main(String[] args) {
      Scanner scnr = new Scanner(System.in);
      int myFirstVar;    // Aligned comments yield less
      int yetAnotherVar; // visual clutter
      int thirdVar;

      // Above blank line separates variable declarations from the rest
      System.out.print("Enter a number: ");
      myFirstVar = scnr.nextInt();

      // Above blank line separates user input statements from the rest
      yetAnotherVar = myFirstVar;        // Aligned = operators
      thirdVar      = yetAnotherVar + 1; 
      // Also notice the single-space on left and right of + and =
      // (except when aligning the second = with the first =)

      System.out.println("Final value is " + thirdVar); // Single-space on each side of +
   }
}
</zyCode>
</zyFigure>
</zyVersion>
