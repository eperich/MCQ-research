<zyHTML id="78e1bc68-d573-08d7-efa4-8d81c8b84bad" subsection="true">
<h3>Dirty data</h3>
</zyHTML>

<zyHTML id="1d468a1a-a20a-cd7c-87ea-823b7e28ded2">

<p>Raw datasets often contain missing, outlier, and duplicate data.
</p>

<ul> 
<li>
<p>
<zyDefn>_Missing data_ is an unknown or inapplicable value. In a database, missing data is represented as <zyICode>NULL</zyICode>. 
</zyDefn><zyVersion version="python">In Python, missing data is represented as <zyICode>NaN</zyICode> (not a number), <zyICode>NaT</zyICode> (not a time), <zyICode>None</zyICode> (an unspecified object), or a blank value.</zyVersion><zyVersion version="nonprog">In other software, <zyICode>NA</zyICode> and <zyICode>NaN</zyICode> are common representations of missing data.</zyVersion>
</p>
</li>

<li>
<p>
<zyDefn>_Outlier data_ is a numeric value that is much larger or smaller than other values in the same feature. 
</zyDefn>Outlier data is usually defined as two or three standard deviations from the feature mean. 
</p>
</li>

<li>
<p>
<zyDefn>_Duplicate data_ are two or more identical instances in a dataset. 
</zyDefn>Duplicate instances are usually erroneous and should be removed. 
</p>
</li>

</ul>

<p>
<zyDefn>Missing, outlier, and duplicate data are collectively called _dirty data_.
</zyDefn>
<zyDefn>A _dirty instance_ and a _dirty feature_ contain dirty data.
</zyDefn>
</p>

<p>Dirty data creates bias and inefficiencies in data analysis. Data scientists may struggle to interpret missing data. Values in erroneous duplicates appear too often and are weighted too heavily. Outliers skew results due to one potentially erroneous value. Consequently, missing, outlier, and duplicate data should be corrected or deleted. 
</p>

</zyHTML>

<h3>Discarding data</h3>
</zyHTML>

<zyHTML id="e993d8bc-1ba9-1392-9d1a-729ae34537f3">

<p>Dirty data may be removed from a dataset by discarding instances, discarding features, or pairwise discarding.
</p>

<p>
<zyDefn>_Discarding instances_, also called _listwise deletion_ or _complete case removal_, removes dirty instances from the dataset.
</zyDefn>Dirty instances are usually discarded when:
</p>

<ul>
<li>
<p>The dirty instances comprise a small percentage of the dataset.
</p>
</li>

<li>
<p>The dirty  instances are random. When missing or outlier values are correlated with values in another feature, discarding dirty instances introduces bias.
</p>
</li>

<li>
<p>Instances are duplicates. Usually, duplicate instances are erroneous, and one instance should be discarded. 
</p>
</li>
</ul>

<p>
<zyDefn>_Discarding features_ removes dirty features that contain a high percentage of missing values, such as 60% or more. 
</zyDefn>Discarding features does not usually apply to outlier data since, by definition, a small percentage of values can be outliers. Discarding features never applies to duplicate data.
</p>

<p>
<zyDefn>_Pairwise discarding_ retains dirty instances for some analyses and discards dirty instances for others. Instances are discarded only when an analysis uses a dirty feature. 
</zyDefn>With pairwise discarding, the total number of instances varies for different analyses, which complicates comparisons and correlations. For this reason, pairwise discarding is not commonly used.
</p>

</zyHTML>

<zyHTML id="74e6a6e8-eca8-2ca4-52a7-a50257398157" subsection="true">
<h3>Imputing data</h3>
</zyHTML>

<zyHTML id="c24bdcc8-92ec-4907-6b1c-8abdb7b1ef8e">

<p>
<zyDefn>_Imputing data_ replaces missing and outlier data with new values.
</zyDefn>Imputing is more complex than discarding but retains all instances and features. Data may be imputed in several ways:
</p>

<p>
<ul>

<li>
<p>
<zyDefn>_Hot-deck_ and _cold-deck imputation_ replace missing and outlier data with a value from a randomly selected instance. In hot-deck imputation, the value is selected from other instances in the same dataset. In cold-deck imputation, the value is selected from a different dataset.
</zyDefn>
</p>
</li>

<li>
<p>
<zyDefn>_Mean imputation_ replaces missing and outlier data with the mean value of the feature. Missing and outlier data are excluded from the computation of the mean.
</zyDefn>
</p>
</li>

<li>
<p>
<zyDefn>_Regression imputation_ replaces missing and outlier data with a value computed from a regression model. In the regression model, the dependent variable is the dirty feature and the independent variables are other features. _Stochastic regression imputation_ introduces uncertainty by adding or subtracting the regression variance to the new value. Regression models are discussed elsewhere in this material.
</zyDefn>
</p>
</li>

</ul>
</p>

<p>Regression imputation is valuable if the dirty feature is highly correlated with other features. If not, mean imputation is commonly used. Hot- and cold-deck imputation were common when less computer power was available to compute mean and regression but are not widely used today.
</p>

</zyHTML>

<h3>Cleaning data with Python</h3>
</zyHTML>

<zyHTML id="7ef477ab-8736-8bc5-0abb-b6517465d8cc">

<p>In Python, three special symbols represent missing values:
</p>

<p>
<ul> 
<li>
<p><zyICode>None</zyICode> represents any missing Python object, such as a string.
</p>
</li>
<li>
<p>NaN represents a missing numeric value. NaN is a <zyICode>NumPy</zyICode> value, specified as <zyICode>numpy.NaN</zyICode>.
</p>
</li>
<li>
<p>NaT represents a missing datetime value. NaT is a <zyICode>pandas</zyICode> value, specified as <zyICode>pandas.NaT</zyICode>.
</p>
</li>
</ul>
</p>

<p>In addition, blank and 0 sometimes indicate a missing value, as in any tool.
</p>

<p>The pandas DataFrame class has methods that identify dirty data and discard and impute values. Important data cleaning methods are described in the table below. The table includes all required parameters and important optional parameters but excludes infrequently used optional parameters. 
</p>

<p>Methods that change data contain an optional <zyICode>inplace</zyICode> parameter. If <zyICode>inplace</zyICode> is True, changes are made in the input dataframe.  If <zyICode>inplace</zyICode> is False, changes are returned in a new dataframe.
</p>
