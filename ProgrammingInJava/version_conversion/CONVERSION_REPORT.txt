============================================================
CONTENT FORMATTING APPLIED
============================================================

Rich content from XML has been preserved for GPT-5-mini analysis:

1. LaTeX Math:
   - Inline: <ilatex>...</ilatex> → $...$
   - Block:  <blatex>...</blatex> → $$...$$

2. Code Blocks:
   - <codeBlock language='python'>code</codeBlock> → ```python\ncode\n```
   - Supports: python, java, c, cpp, text

3. Inline Code:
   - <code>text</code> → `text`

4. Images:
   - <image alt="description"> → [Image: description]

5. HTML Tags:
   - Preserved: <p>, <i>
   - Line breaks: \n for newlines within fields

6. Special Characters:
   - Properly escaped for CSV format
   - Fields containing commas, quotes, or newlines are quoted


============================================================
CSV FORMAT
============================================================

Each CSV file contains:
- Header row: text,answer,a,b,c,d,e,f,g,h,hint,feedback
- One row per question
- Columns a-d: The 4 answer choices
- Columns e-h: Empty (reserved for future use)
- hint column: Empty (no hints in source data)
- feedback column: Empty (no feedback in source data)
